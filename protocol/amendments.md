# Protocol Amendments

Record of changes to the protocol, as required by PRISMA 2020.

| Date | Section Changed | Description of Change | Justification |
| --- | --- | --- | --- |
| 2026-03-23 | Section 7 Extraction | Added `NR` for not reported and `NA` for not applicable fields, mainly affecting cost and efficiency extraction. | The extraction pilot showed that many papers omit cost fields, requiring an explicit distinction between absence of data and inapplicability. |
| 2026-03-23 | Section 7 Extraction | Added a rule for `ameacas_validade`: infer from reported limitations when the paper has no formal threats-to-validity section. | ML and AI-agent papers rarely use empirical-SE threats-to-validity sections, so extraction needed a consistent inference rule. |
| 2026-03-23 | Section 6 Screening | Added a CI1 guide: abstracts mentioning "long-term memory" or "experience" without clear cross-session persistence are classified as maybe and verified in full text. | Screening pilot cases showed that CI1 often cannot be determined from abstracts alone. |
| 2026-03-25 | Section 5 Search Strategy | Refined search strings after audit: added `code assistant`, `agentic coding`, `SWE agent`, `persistent memory`, `persistent context`, and `cross-task`; corrected arXiv/Scopus asymmetry; added title-field search for key C1 terms. | Audit against comparable SLRs, known systems, and alternative term formulations improved seed-paper coverage and reduced unexplained database asymmetry. |
| 2026-03-26 | Sections 3 and 6 Eligibility and Screening | Completed the second criteria stress test before large-scale screening: clarified explicit versus parametric persistence, runtime versus training-time evidence, agent versus human-authored artifacts, minimum evaluation floor, short-paper handling, intra-session exclusions, generic-agent inclusion when SE evaluation exists, duplicate handling, borderline cases, and screening flowchart. | The stress test identified grey zones around generic LLM-agent papers, evaluation sufficiency, and persistence interpretation that needed deterministic screening guidance. |
| 2026-04-24 | Section 3 Eligibility | Added the executable-self-evolution boundary rule: systems that externalize procedural state into executable artifacts during task execution may be retained as boundary architecture evidence when the artifact is created, invoked, and reused by the agent during task execution. | External review showed that task-local executable artifacts needed a named boundary rule so they would not be silently described as cross-session evidence. |
| 2026-04-25 | Section 7 Extraction | Corrected EET retrieval coding from BM25 to TF-IDF and added `tf_idf` to the controlled vocabulary. | Direct verification against the digest and source text showed that EET uses TF-IDF top-1 retrieval. |
| 2026-04-28 | Section 5 Search Strategy | Finalized the three-family search design. The arXiv category set is `cs.SE OR cs.AI OR cs.CL OR cs.LG`. The Scopus search remains branch-split because the combined query exceeded API limits; Family C uses a Scopus-specific narrowed translation after burden calibration. | The final search-string gate accepted the design after category, hyphenation, Scopus-asymmetry, and execution-metadata risks were checked. |
| 2026-05-08 | Reporting and Release | Canonicalized the current corpus artifact names and release plan. The private workspace uses unsuffixed source-of-truth files, and the public replication package will be released as `2.0.0` with a new version-specific DOI. | The current review is the only reportable corpus. Unsuffixed file names reduce denominator drift, and a version-specific release avoids tying the current package to obsolete public metadata. |
| 2026-05-12 | Sections 3 and 6 Eligibility and Screening | Narrowed CI4 from English-or-Portuguese eligibility to English-only eligibility in the protocol, screening guide, search strategy, and manuscript reporting. | The target manuscript and venue strategy are English-language, no screened record depended on Portuguese-language eligibility, and the refinement improves reporting clarity without changing corpus membership, exclusion counts, or synthesis denominators. |
